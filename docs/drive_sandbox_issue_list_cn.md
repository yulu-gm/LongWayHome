# DriveSandbox 问题清单

日期：2026-05-15
来源：`Task 0.1` 基线验证、Godot 烟测、截图 `tmp/drive_sandbox_smoke.png`、两张强制视觉参考。

## 必须修 / 必须推进

### 1. 当前场景仍是空测试场

- 现象：只有大平面、直路、中心线和原型车。
- 影响：无法接近参考图里的公路旅行氛围。
- 方向：后续道路任务必须加入路肩、护栏、树、岩石、标志牌、地形边界和 POI 入口。
- 优先级：P0。

### 2. 车辆缺少“旅行车是家”的视觉身份

- 现象：当前车是红色盒状低模，不能表达老旧旅行车、车顶行李、后备箱生活空间。
- 影响：削弱核心体验“车辆是家”。
- 方向：优先做老旧旅行车轮廓、车顶架、行李箱、备用胎等可读外观。
- 优先级：P0。

### 3. 调试 HUD 不是玩家 HUD

- 现象：左侧面板占屏较大，显示大量调参和遥测信息。
- 影响：适合开发，不适合正式驾驶体验。
- 方向：Task 2.1 应新增玩家驾驶 HUD，保留 debug HUD 但不作为默认体验。
- 优先级：P0。

### 4. 缺少时间与天气视觉层

- 现象：当前只有单一白天/灰绿环境，没有雨夜、傍晚、雾天、湿路反光。
- 影响：无法承载参考图里的天气时间矩阵。
- 方向：先做白天晴、傍晚、夜雨、雾天四套基础氛围。
- 优先级：P1。

### 5. 截图烟测不能在 headless 模式稳定完成

- 现象：`capture_drive_sandbox.gd` 在 `--headless` 下会卡住，非 headless 可成功保存截图。
- 影响：自动化视觉回归不够稳定。
- 方向：后续建立独立截图/渲染验证方案，或固定使用非 headless console 渲染路径。
- 优先级：P1。

## 可调参

### 6. 驾驶手感需要人工长测

- 现象：自动烟测只证明加速、左右转向和手刹流程可执行，不能证明长时间手感。
- 影响：漂移、抓地、高速稳定、碰撞成本仍需人工体验。
- 方向：在 Task 0.2 之后的驾驶相关改动中保留手动长测清单。
- 优先级：P1。

### 7. 相机已有 2.5D 基础，但缺少路况展示调校

- 现象：相机能看到车辆和道路，但当前道路太空，无法判断构图是否适合复杂环境。
- 影响：后续加道路/POI 后可能需要调整 offset、look-ahead 和缩放。
- 方向：等道路资产密度上来后再做相机调参。
- 优先级：P2。

## 暂不处理

### 8. 城市区域、行人和复杂交通

- 原因：MVP 优先长途公路、休息站、小镇节点，不先做完整城市道路。
- 触发时机：Milestone 8 之后或城市路线进入垂直切片时。

### 9. 高精度车辆内饰

- 原因：当前优先外部驾驶视角和核心循环；车内氛围可在车内事件、电台、旅馆休息之后推进。
- 触发时机：车内事件或休息/电台系统开始时。
