# 公路驾驶沙盒设计

## 背景

当前项目是一个几乎空白的 Godot 4.6 .NET 工程。目标是做一个以车辆驾驶为核心的公路游戏：玩家使用 `W/S/A/D` 控制车辆，`Shift` 手刹，画面采用 2.5D 斜俯视视角，类似高位 45 度观察车辆和道路的公路驾驶视角。

本设计分两步推进：

1. 第一阶段先做驾驶沙盒，把车辆物理、手刹漂移、悬挂、引擎、变速箱、驱动形式和调参验证做扎实。
2. 第二阶段再加入道路生成和动态相机，不引入关卡、任务、失败条件等玩法结构。

## 阶段边界

### Phase 1：驾驶物理沙盒

第一阶段只关注车辆是否真实可玩。交付物是一个可运行的 `DriveSandbox` 场景，包含：

- 一辆可驾驶的原型车。
- 2.5D 斜俯视相机。
- 测试地面和基础路面。
- C# 自研车辆物理核心。
- 可切换车辆 preset 的运行时面板。
- 显示速度、RPM、档位、滑移和轮胎载荷的调试 HUD。

第一阶段不做道路生成、关卡、任务、失败条件、美术完整性和手柄输入。

### Phase 2：无关卡公路生成层

第二阶段在车辆核心稳定后加入：

- 动态道路片段生成。
- 弯道、路宽和路面摩擦区域。
- 动态相机的速度缩放、前瞻和跟随平滑。

第二阶段仍不做关卡概念。道路系统只提供碰撞地面、路面摩擦参数和视觉环境，不重写第一阶段车辆核心。

## 视角与场景目标

目标视角不是正顶视，而是 2.5D 斜俯视/近似等距视角：

- 相机高位倾斜观察车辆。
- 车辆能看到车顶和部分侧面体积。
- 第一阶段相机可以先保持固定角度跟随车辆。
- 第二阶段再加入速度相关缩放、前瞻偏移和更细的平滑策略。

## 车辆物理方案

车辆物理采用 C# 自研方案，不使用 Godot 内置 `VehicleBody3D` 作为核心。

主结构：

- `RigidBody3D` 作为车身，承载质量、惯性、碰撞和力矩。
- 四个自研 `RaycastWheel` 负责接地、悬挂和轮胎力。
- `Powertrain` 模拟发动机、RPM、离合近似、变速箱和扭矩输出。
- `Differential` 根据 FWD / RWD / AWD 分配轮端扭矩。
- `TireForceModel` 计算纵向牵引/制动、横向抓地、滑移和手刹影响。
- `VehicleTelemetry` 给调试 HUD 和面板输出数据。

第一版轮胎模型使用可调近似模型，不直接上完整 Pacejka 公式。原因是第一阶段需要快速调出可控、可读、能漂移的拟真手感；可调抓地/滑移曲线比复杂公式更适合早期沙盒。

## 手刹漂移

`Shift` 手刹不是单纯增强刹车，而是进入后轮漂移状态：

- 后轮获得额外制动力。
- 后轮横向抓地降低。
- 抓地降低和恢复使用曲线或平滑插值，避免松手刹时突然抽回。
- HUD 显示后轮滑移，方便判断漂移是否来自正确的轮胎状态。

## 多车辆基准

车辆核心必须支持多种基准车，而不是写死某一种后驱车。

第一阶段至少提供四个 preset：

- 前驱家用车：稳定、推头明显、漂移困难。
- 后驱标准车：中性基准，手刹和油门配合可漂移。
- 四驱公路车：抓地强，出弯稳定，滑移恢复快。
- 后驱跑车：动力强，后轮更容易突破抓地，漂移更容易维持。

Preset 参数包括：

- 质量、重心、轴距、轮距。
- 悬挂弹簧、阻尼、行程。
- 轮胎抓地、滑移峰值、恢复速度。
- 发动机扭矩曲线、怠速/红线 RPM。
- 档位比、终传比、换挡阈值。
- FWD / RWD / AWD 驱动形式和扭矩分配。

## 输入与运行时面板

输入：

- `W`：加速。
- `S`：刹车，低速时进入倒车。
- `A/D`：转向。
- `Shift`：手刹。
- `Tab`：显示/隐藏运行时面板。

运行时面板：

- 切换车辆 preset。
- 微调当前运行实例的关键参数：发动机扭矩倍率、轮胎抓地、刹车力度、手刹后轮抓地损失、悬挂弹簧和阻尼。
- 调参只影响运行中的实例，不直接污染默认 preset。

HUD：

- 速度。
- RPM。
- 档位。
- 当前驱动形式。
- 手刹状态。
- 前后轮滑移。
- 四轮悬挂压缩和轮胎载荷。

纯计算模块尽量不依赖 Godot scene，这样可以用 C# 单元测试覆盖动力、差速和轮胎模型。

## 验收标准

Phase 1 完成必须满足：

- `DriveSandbox` 能作为项目主场景运行。
- `W/S/A/D/Shift` 驾驶完整可用。
- 车辆不会在正常驾驶中明显数值爆炸或无限加速失控。
- 四轮悬挂随载荷和地面接触产生压缩，并影响轮胎力。
- RPM、档位和扭矩输出会随油门、速度、档位变化。
- `Shift` 能诱发后轮漂移，松开后能逐步恢复抓地。
- FWD / RWD / AWD / RWD 跑车四个 preset 的手感差异可明显感知。
- 运行时面板能切换 preset 并显示遥测。
- 自动化测试覆盖动力系统、差速分配、轮胎抓地和手刹抓地变化。

## 风险与约束

- 自研车辆物理比 `VehicleBody3D` 代码量更大，但更适合长期扩展。
- Godot 物理调参需要真实运行验证，不能只靠编译和单元测试。
- 第一阶段的视觉资产只做可读占位，不把美术品质作为验收阻塞。
- 当前目录不是 git 仓库，因此设计文档和计划无法在本目录内 commit。
