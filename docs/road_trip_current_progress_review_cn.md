```text
驾驶沙盒

→ 驾驶 HUD
→ 路线与程序化道路
→ 地点服务
→ 下一路线选择
→ 事件数据、导演、事件面板、12 个 MVP 事件
```

## 当前已经具备的能力

### 驾驶与道路

- `DriveSandbox.tscn` 仍是主入口。
- 原型车、2.5D 相机、基础驾驶输入和 debug 面板可用。
- 车辆状态已能影响驾驶参数或风险提示。
- 程序化道路已支持连续路段、路肩、标线、树、岩石、电线杆、护栏、路牌等低模占位。
- 道路切线采样已修正，黄线、白线和路边物会跟随道路方向。

### 旅途状态与资源

- 已有 `TripState` 作为旅途状态单一真值。
- 已有 `ResourceLedger` 处理燃油、精力、碰撞损伤等结算。
- 已有低资源提示逻辑，能区分低油、低耐久、低精力、雨天低轮胎、夜晚低车灯等风险。

### HUD 与服务 UI

- 已有玩家驾驶 HUD，显示时间、天气、燃油、耐久、精力、金钱、目标距离、速度和提示。
- Debug HUD 已和玩家 HUD 分离。
- 已有 `ServicePanel`，支持加油、住宿、维修、购买基础补给、短暂休息。
- 服务前会显示资源变化预览；金钱不足或服务不可用时会给出禁用原因。

### 路线与地点循环

- 已有 `RouteGraph` / `RouteNode` / `RouteChoice`。
- 当前地点可以生成 2 到 3 条路线。
- 路线包含目的地、距离、道路类型、风险、天气倾向、服务、事件标签和路段 id。
- 已有 `RouteSelectionPanel`。
- 已有 Godot smoke 覆盖连续两次“服务 → 路线选择 → 回到驾驶”的循环。

### 事件系统

- 已有事件数据结构：事件 id、标题、文本、分类、权重、冷却、一次性标记、触发条件、选项和结果。
- 已有 `RoadEventSelector` 处理 eligible events、冷却、一次性事件和权重选择。
- 已有 `RoadEventDirector` 根据低油量、雨夜、疲劳等状态调整候选事件权重。
- 已有 `EventChoicePanel`，能显示事件文本和选项，选择后更新 `TripState` 并回到驾驶。
- 已有 12 个 MVP 事件内容：
  - 抛锚车辆
  - 搭车者
  - 路边商贩
  - 临时施工
  - 雨夜封路
  - 油价上涨传闻
  - 废弃小镇传闻
  - 轮胎异响
  - 车灯闪烁
  - 深夜电台
  - 汽车旅馆陌生人
  - 旧磁带

## 真实缺口与风险

### 1. 已有循环骨架，但还不是完整可玩垂直切片

当前已经能验证“服务 → 路线选择 → 回到驾驶”的循环，也有事件数据和事件面板；但距离完整 MVP 仍缺：

- 物品/背包/车辆存储。
- 真实地图面板。
- 天气昼夜系统。
- 存档。
- 调参后的 10 到 15 分钟试玩路径。

### 2. 事件系统已成型，但还需要更深的运行时接线

当前事件定义、导演、面板、12 个事件内容都已完成；后续还需要把事件更自然地接入实际驾驶过程：

- 驾驶中按路段/距离/状态触发事件。
- 抵达地点时触发地点事件。
- 事件结果写入长期标记，并影响后续路线或地图。

### 3. 视觉仍主要是程序化占位

已经不再是纯空白测试场，但很多关键画面资产仍是缺口：

- 老旧旅行车主角资产。
- 加油站、汽车旅馆、修理铺、休息区等 POI 建筑。
- 地图底图、路线曲线和节点图标。
- 事件头像、人物、事件插图和路边事件道具。
- 雨夜、雾天、湿路反光、车灯光锥和音频氛围。

这些已记录在 `docs/road_trip_visual_asset_gaps.md`，后续每个视觉任务都应继续对照更新。

### 4. 工作区注意事项

当前有一批 Godot 自动生成的 `.uid` 未跟踪文件。它们不是本进度文档的内容，后续提交时应继续显式暂存任务相关文件，避免把生成文件误带入提交。
