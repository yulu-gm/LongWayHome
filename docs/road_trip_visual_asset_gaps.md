# 公路旅行游戏视觉资产缺口清单

此文件用于记录“当前资源无法逼近 `design.png` 参考效果”的位置，以及可替换资产建议。

## 当前状态

`Task 0.1` 基线验证后，已确认当前沙盒缺少多类逼近参考图所需资产。`Task 3.3` 已加入程序化道路生成、路肩、标线、树、岩石、电线杆、护栏、路牌、河边色块等低模占位；`Task 3.3a` 已修正道路切线采样，让黄线、白线和路边生成物跟随道路方向，并加宽两侧绿化带。`Task 4.2` 已加入深色半透明地点服务面板，`Task 4.3a` 已加入路线选择列表入口，但地点/服务/地图相关视觉仍是 UI 占位。后续每个影响画面的任务都必须继续检查并更新本文件。

## 老旧旅行车主角资产

- 用途：承载“车辆是家”的核心体验，作为玩家长期驾驶和改装对象。
- 参考图对应：老旧旅行车、车顶行李架、备用轮胎、后备箱/车内储物。
- 当前缺口：当前车辆是红色盒状原型车，没有旅行车轮廓、旧车细节和载物特征。
- 临时替代：用低模车身加车顶箱、备用胎、行李箱、工具箱等几何体拼出轮廓。
- 推荐资产：低多边形旧旅行车 / station wagon，支持车顶架、后备箱可见储物、可换色材质。
- 优先级：P0

## 道路环境模块

- 用途：把直线测试道路扩展成可巡航、可发生事件的公路。
- 参考图对应：郊外公路、山区道路、高速公路、沙漠道路、小镇街道。
- 当前缺口：`Task 3.3a` 已让程序化沥青路、路肩、中心黄线、边线、简易护栏、路牌和电线杆统一跟随道路切线，并把两侧绿化带加宽；仍缺高质量弯道路面模块、道路磨损贴花、真实路口/匝道、小桥、坡度和雨夜湿路材质。
- 临时替代：继续用 Godot 基础网格制作路肩、护栏、限速牌、方向牌、路锥和岩石；用程序化路段控制道路长度、回收和预加载。
- 推荐资产：低多边形公路模块包，包含直路、弯道、十字路口、护栏、标志牌、路面贴花。
- 优先级：P0

## 自然环境与路边细节

- 用途：提供参考图中的森林、山路、河流、岩石、草地、沙漠等旅行氛围。
- 参考图对应：郊外公路、山区道路、沙漠荒野、环境设计参考。
- 当前缺口：`Task 3.3` 已加入低模松树、岩石、灌木和河边色块占位，画面不再是纯空地；仍缺成组草丛、地形起伏、远山/河岸过渡、沙漠/雪地专属植被和更高密度的路边小物。
- 临时替代：继续使用低模松树、灌木、岩石、草块、仙人掌、枯树等简单 Mesh，并按道路标签组合生成。
- 推荐资产：低多边形自然环境包，包含 conifer trees、rocks、grass clumps、river edges、desert props。
- 优先级：P0

## 路边 POI 建筑

- 用途：支撑加油、住宿、维修、事件和路线选择。
- 参考图对应：加油站、汽车旅馆、修理厂、休息区、路边摊、废弃建筑。
- 当前缺口：没有任何可停靠地点或建筑资产。
- 临时替代：用基础几何搭建 motel 招牌、加油泵、修理厂门面和休息区亭子。
- 推荐资产：低多边形 roadside POI 包，包含 motel、gas station、repair garage、diner/stall、abandoned house。
- 优先级：P0

## 天气与时间渲染资产

- 用途：表现白天、傍晚、夜晚、雨天、雾天等核心情绪变化。
- 参考图对应：天气与时间变化示例、高速雨夜、夜晚城市、环境氛围参考。
- 当前缺口：只有单一光照和材质，没有雨、雾、湿路反光、车灯光锥、尾灯反射。
- 临时替代：使用 DirectionalLight/WorldEnvironment 调色、简单粒子雨、半透明雾层、发光尾灯材质。
- 推荐资产：Godot 4 可用天气 VFX、低成本雨粒子、雾体积、湿路材质、headlight/taillight shader。
- 优先级：P1

## 玩家驾驶 HUD 图标与样式

- 用途：替换调试面板，形成参考图中的驾驶 HUD。
- 参考图对应：深色半透明 HUD、琥珀色重点信息、油量/耐久/精力/时间/天气/速度。
- 当前缺口：`Task 2.1` 已加入底部左侧玩家 HUD、深色半透明面板、琥珀色状态条和核心旅途数值；仍缺少正式图标、仪表盘式速度表、天气图标和更精致的像素/低多边形 HUD 美术。
- 临时替代：继续使用 Godot UI 绘制深色半透明面板、文字、基础进度条和简短中文状态。
- 推荐资产：小型 HUD 图标集，包含 fuel、wrench/durability、mood/energy、weather、clock、map marker、money。
- 优先级：P1

## 地点服务面板与服务图标

- 用途：支撑抵达加油站、汽车旅馆、修理铺和休息区后的服务选择，让“停车补给”成为旅途呼吸点。
- 参考图对应：`design.png` 的“休息站 / 汽车旅馆”“改装与维修”界面，以及第二参考图中的汽车旅馆、加油站、修理厂、休息区等可互动 POI。
- 当前缺口：`Task 4.2` 已加入右侧深色半透明 `ServicePanel`、中文服务列表、资源变化预览和不可用原因；仍缺正式服务图标、地点背景图、地点招牌/窗口暖光/加油泵/维修铺外观，以及和真实 POI 停靠状态绑定的画面表现。
- 临时替代：继续使用 Godot UI 面板、文字、按钮和琥珀色强调；服务点本身暂由路线目的地数据和面板标题表示。
- 推荐资产：低多边形 motel/gas station/repair garage/rest area 模型包，小型服务图标集，地点背景或停车场插画式 UI 背板。
- 优先级：P1

## 路线选择地图 UI

- 用途：让玩家在服务点后根据目的地、距离、风险、天气和服务决定下一段旅途。
- 参考图对应：`design.png` 的“地图与导航”界面，以及第二参考图中节点化路线、服务点和传闻标记方向。
- 当前缺口：`Task 4.3a` 已加入右侧深色半透明 `RouteSelectionPanel` 列表，显示目的地、距离、道路类型、风险、天气和服务；仍缺真实地图底图、路线曲线、节点图标、传闻/服务标记、当前车位与目标方向的视觉关系。
- 临时替代：继续使用列表式路线卡片承载路线选择，并用中文风险/天气/服务标签保证决策可读。
- 推荐资产：低多边形或手绘风地图底图、路线节点/服务/风险/天气小图标、可复用的路线高亮线材质。
- 优先级：P1
